# JVT Agent Registry

This directory defines the canonical list of JVT internal agents.

Each JSON file represents one operating role, not one temporary task.

The purpose is to give the Mac mini control plane a stable source of truth for:

- which agents exist
- what each agent owns
- which queues and data sources it touches
- whether it is autonomous, review-driven, or mostly human-owned

These files are not execution logs.
They are role manifests that the control panel and future orchestrator logic can read.

## Executive Operations Manager

`executive-ops-manager.json` is the manifest for the EOM role. It owns the
single next-action brief generated by:

```bash
python3 ops/agent-control/eom_agent.py
```

The EOM can rank, stage, brief, and escalate. It cannot spend, send, contact
third parties, submit applications, trade live, move funds, mine, stake, create
wallets, or make external commitments.

## Content Operations

`content-ops.json` is the manifest for YouTube/Instagram content packet prep.

The content agent may draft scripts, captions, thumbnail briefs, and approval
checklists from existing JVT proof assets. It must stop before:

- posting
- account setup
- API token use
- paid scheduling tools
- scraping
- comment/DM automation
- fake engagement

## Local Task Runner

`local-task-runner.json` is the manifest for the allowlisted local executor.

Queue:

- `ops/agent-control/tasks/pending`
- `ops/agent-control/tasks/completed`
- `ops/agent-control/tasks/held`
- `ops/agent-control/tasks/failed`

Refresh command:

```bash
python3 ops/agent-control/local_task_runner.py
```

Daily seed path:

```bash
python3 ops/agent-control/growth_ops_checkin.py
```

Growth check-in seeds the standard internal task set once per day, then invokes
the local task runner. The runner's LaunchAgent also wakes periodically to
process any pending task files added by a human or another approved internal
agent.

The runner does not execute arbitrary shell from task files. It only supports
hardcoded internal task types and holds anything that mentions approval-gated
actions such as sending, spending, posting, live trading, wallets, mining,
staking, applications, registrations, or deletion.
