# JVT First-Client Readiness Plan

This is the practical sequence for moving JVT from internal setup to first real paying clients.

This is an operating guide, not legal advice.

## Current Position

Already live on the Mac mini:

- public website
- control panel
- lead pipeline
- reviewed outreach workflow
- inbound mailbox listener
- local private-doc-intel demo
- Tailscale remote operator access

## Legal And Commercial Gate Before Wider Sending

If JVT is operating from New Jersey, the cleanest practical path is:

1. choose the short-term structure
2. register the name you plan to use publicly
3. get the tax IDs and banking setup that match that structure
4. only then widen outbound activity

### Short-Term Decision

Pick one of these and act consistently:

- sole proprietorship under your legal name for immediate testing
- sole proprietorship using a registered trade name
- single-member LLC if you want a cleaner liability and banking story before growth

### Practical Rule

Do not widen commercial outreach under `JVT Technologies` until the business name and bankable structure match the way you present yourself publicly.

## Official Source Reminders

Use these official sources when completing the paperwork:

- SBA business structure and registration guidance
- IRS EIN guidance
- NJ business registration and trade-name guidance
- FTC CAN-SPAM guidance for commercial email

## Recommended Order

### Phase 1: 1 to 3 days

- decide sole proprietor now vs single-member LLC now
- if using `JVT Technologies` publicly in New Jersey, register the name appropriately
- get EIN after the structure is set
- open business banking once the name and EIN are aligned
- set a valid postal address policy for commercial email compliance

### Phase 2: 1 to 3 days

- keep the control panel and doc-intel backend running as launchd services
- initialize the local client workspace under `~/Documents/JVT-Technologies`
- create the first client workspace template path
- confirm the mailbox listener and outbound review flow are healthy

### Phase 3: 3 to 7 days

- review the public site copy to ensure it matches the real legal/business state
- keep the free fit call as the entry point
- keep the first paid offer simple:
  - $500 discovery
  - $3,500 to $5,000 pilot
- prepare one invoice template and one proposal template

### Phase 4: 7 to 14 days

- build one curated tranche of 25 high-fit leads
- select 5 to 8 for the first reviewed wave
- send only manually reviewed messages
- log every send, reply, and follow-up in the existing queue model

### Phase 5: 14 to 30 days

- run follow-up on the first wave
- book discovery calls
- convert one discovery into one scoped pilot
- collect feedback from the first real workflow
- tighten the offer around what prospects actually ask for

## Sending Rules

- no bulk send engine
- no false personalization
- no misleading subject lines or sender identity
- include a valid postal address and unsubscribe path for commercial email
- keep reviewed sends low volume until the legal/business basics are clean

## First Client Goal

The first win is not scale.

The first win is:

- one paying discovery or pilot
- one clean delivery folder
- one repeatable intake and billing flow
- one proof point that the private-AI story solves a real workflow problem
