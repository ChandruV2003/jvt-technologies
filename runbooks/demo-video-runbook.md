# Demo Video Runbook

## Goal

Record a short, polished, honest video that shows the system working with real local behavior.

Target length:

- 60 to 120 seconds for a teaser
- 2 to 4 minutes for a fuller walkthrough

## Recording Setup

- use the backend-served demo UI
- use the prepared sample documents
- use a clean browser window with unnecessary apps hidden
- keep the question set short and repeatable

## Recommended Sequence

1. title card or verbal intro
2. quick website/homepage frame
3. switch to the operator UI
4. upload a sample document set
5. ask a grounded billing or confidentiality question
6. highlight the answer and citations
7. close with privacy/deployment positioning

## Best Take Sequence

1. show the homepage hero for 4 to 6 seconds
2. jump to the operator UI
3. upload the three recommended sample documents
4. ask the billing-dispute question in `mlx-local` mode
5. pause on the cited answer
6. point to one citation and source snippet
7. end on the deployment/privacy explanation

## Recording Guidance On This Mac

- close unrelated windows before recording
- set browser zoom so citations are readable
- avoid tiny pointer movements
- pause briefly before each click so editing is easier later
- keep the mouse path simple and intentional

## Intro Text

“JVT Technologies builds private AI systems for document-heavy businesses. This demo shows a local-first document assistant that answers from uploaded documents and cites its sources.”

## Outro Text

“The system is designed for controlled internal use, grounded answers, and deployment options that fit privacy-sensitive teams.”
