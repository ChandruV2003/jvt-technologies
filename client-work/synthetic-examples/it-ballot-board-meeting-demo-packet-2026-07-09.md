# Synthetic Demo Packet: IT Ballot + Board Meeting Workflow

Generated: 2026-07-09

Status: synthetic internal sales/demo artifact. This does not represent a real election, real association, or legal/compliance guidance.

## Demo Purpose

Show how JVT can help an IT consulting / election-support vendor turn messy board-meeting and ballot-administration work into reviewed packets, checklists, and audit-friendly task queues.

## Synthetic Customer

Harbor View Condominium Association is preparing for an annual board election. The vendor supports meeting A/V, registration, ballot handling, tabulation support, and post-meeting documentation.

## Workflow Pain

- Requirements arrive across email, PDFs, meeting notes, and phone calls.
- Staff manually builds the candidate list and meeting checklist.
- Ballot packets need consistent formatting and review.
- Board-meeting action items get lost after the event.
- Sensitive election work needs human approval and audit notes, not black-box automation.

## JVT Workflow Layer

```text
Client emails / meeting notes / uploaded docs
  -> intake classification
  -> missing-info checklist
  -> ballot packet draft
  -> board-meeting run-of-show
  -> staff review and approval
  -> post-meeting action packet
  -> audit log
```
